# SECTION-TEMPLATES.md — 레이아웃 축 (섹션 구조 · 슬롯)

> 이 문서는 **2축 디자인 시스템** 중 **레이아웃·구조 축**이다.
> 다른 축(타이포·스페이싱·컬러)은 `./DESIGN-RULES.md` 참조.
>
> ⚠️ **여기엔 실제 섹션을 미리 만들어 두지 않는다.** 템플릿(슬롯 구조)만 기술한다.
> 실제 섹션은 현빈 요청 시 PM이 `build_basic_section` 등으로 생성한다.
> 역할별 템플릿(Hook/Main/Detail/CTA 세부)은 **추후 업데이트 예정**.

---

## 0. 2축 개념

```
                 레이아웃 축 (이 문서)
                 = 어떤 슬롯이 어떤 순서로
                          │
   스타일 축 ─────────────┼──────────────  하나의 섹션
   (DESIGN-RULES.md)      │
   = 슬롯에 적용할        │
     크기·갭·컬러         ▼
```

- **레이아웃 축**: 섹션이 어떤 **슬롯**(라벨/메인카피/본문/에셋 …)을 어떤 순서로 갖는가.
- **스타일 축**: 각 슬롯의 폰트 크기·갭·컬러·정렬.
- PM은 이 둘을 곱해서 한 섹션을 조립한다.

---

## 1. 슬롯 정의 (역할별)

| 슬롯 | 역할 | 블록 type | 필수 |
|------|------|----------|------|
| `label` | 섹션 키워드/카테고리 (작게) | `label` | 선택 |
| `mainCopy` | 핵심 한마디 (대제목) | `h1` | **필수** |
| `body` | 설명/카피 | `body` | 선택 |
| `asset` | 이미지 자리 (비율 프리셋) | asset preset | 선택 |

> 슬롯은 의미 단위다. 한 슬롯이 비면 그 갭도 함께 생략한다.

---

## 2. 기본 섹션 템플릿 (BASIC)

가장 자주 쓰는 단일 섹션. `build_basic_section` 도구가 이 템플릿을 그대로 조립한다.

```
[상단 패딩]
label      (선택)
mainCopy   (필수, h1)
body       (선택)
asset      (선택, 기본 img1)
[하단 패딩]
```

- 슬롯 채움 규칙: **메인카피와 본문 + 에셋 1개**가 기본. 라벨은 필요할 때만.
- 갭·크기·정렬은 `DESIGN-RULES.md`의 표준 패턴(상하 100 / 라벨50 / 본문30 / 에셋50, center)을 따른다.
- 도구 매핑: `build_basic_section({ mainCopy, body?, label?, assetPreset?, align? })`.

### 에셋 비율 프리셋
| preset | 용도 |
|--------|------|
| `img1` | 단일 와이드 이미지 (기본) |
| `img2` | 2분할 |
| `img3` | 3분할 |
| `text-img` | 텍스트 + 이미지 좌우 |

## 4. PM 사용 가이드

1. 요청이 **"기본 섹션"** / **"~한 내용으로 섹션 만들어줘"** → `build_basic_section` 1콜.
2. 프로젝트 맥락(메인컬러/타깃/USP, `project.meta.json`)이 부족하면 → **디폴트 섹션**(메인카피+본문+에셋)으로 생성하고 카피는 무난하게.
3. **여러 섹션** 요청 → 섹션별로 `build_basic_section` 반복.
4. 세밀 조립이 필요하면 `add_section` → `add_text_block` → `add_asset_block`.
5. 역할별 톤·금칙어는 `DESIGN-RULES.md`의 `AVOID`/`MOTIF`가 채워지면 그때 반영.
