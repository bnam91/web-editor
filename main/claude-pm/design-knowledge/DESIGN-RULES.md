# DESIGN-RULES.md — 스타일 축 (타이포 · 스페이싱 · 컬러)

> 이 문서는 **2축 디자인 시스템** 중 **스타일 축**이다.
> 다른 축(레이아웃·구조)은 `./SECTION-TEMPLATES.md` 참조.
> PM은 섹션을 만들거나 다듬을 때 이 두 문서를 함께 적용한다.
>
> ⚠️ **작성 현황**: 입력된 토큰만 채워져 있음. `AVOID`(피해야 할 표현)와
> `MOTIF`(브랜드 모티프/무드)는 현빈이 아직 지정 전 → **추후 업데이트 예정**.
> 그 외 값도 실측 레퍼런스 기반 초안이며 프로젝트별로 조정될 수 있다.

## 1. 타이포그래피 위계

블록 역할별 기본 스케일. px는 `build_basic_section` / `add_text_block` 기본값과 정렬.

| 역할 | type | 크기(px) | weight | letter-spacing | line-height | 비고 |
|------|------|---------|--------|----------------|-------------|------|
| 라벨 | `label` | 24–28 | 700 | +1.5px | 1.2 | 메인카피 위 작은 키워드. 대문자 권장(TBD) |
| 메인카피(대제목) | `h1` | **90–100** | 600 | 약 −2% | 1.1 | 섹션의 핵심 한마디 |
| 소제목 | `h2` | 40–56 | 600 | −1% | 1.2 | |
| 보조 제목 | `h3` | 30–40 | 500 | 0 | 1.3 | |
| 본문 | `body` | **28–32** | 300 | 0 | 1.6 | 설명/카피 |
| 캡션 | `caption` | 16–20 | 400 | 0 | 1.5 | 부연·주석 |

### 위계 원칙
- **메인카피 : 본문 ≈ 3:1 이상** (예: 100px ↔ 30px). 위계가 약하면 메인카피를 키우거나 본문을 줄인다.
- 한 섹션에 메인카피(h1)는 1개. 여러 개 필요하면 섹션을 나눈다.
- weight는 **제목 무겁게(600+) / 본문 가볍게(300)** 대비로 위계를 보강한다.

### 실측 레퍼런스 (live exemplar)
현재 프로젝트의 `sec_wd3nixu`가 기준 섹션. 실측 토큰:
- label 26px / 700 / +1.56px
- 메인카피(h1) 104px / 600 / −2.08px / lh 1.1
- body 36px / 300 / lh 1.6
- 모든 텍스트 **center** 정렬

## 2. 스페이싱 (갭블록)

세로 리듬은 **갭블록(gap-block) 높이**로 제어한다.

- **단위**: 20px부터 **10px 단위** (20, 30, 40, 50, 60 … 100).
- **상하단 패딩**: 섹션 위·아래 여백도 갭블록으로 처리 → 기본 **100**.

### 기본 섹션 표준 갭 패턴
`build_basic_section`이 자동 적용하는 순서:

```
[상단 패딩 100]
라벨
[갭 50]
메인카피(h1)
[갭 30]
본문(body)
[갭 50]
에셋(이미지 자리)
[하단 패딩 100]
```

- 라벨 → 메인카피: **50**
- 메인카피 → 본문: **30** (가까이 붙여 한 덩어리로)
- 본문 → 에셋: **50**

## 3. 컬러

> **테스트 단계**: 현재 컬러 카테고리는 **Apple 시스템 컬러**를 임시 기준으로 사용.
> 브랜드 확정 시 교체 예정 (추후 업데이트).

### 컬러 변수 (프로젝트 스코프)
컬러는 좌측 **디자인시스템 패널**의 컬러 변수로 관리. 블록은 `var(--color-<name>, #fallback)`로 바인딩되어 변수 변경 시 일괄 반영(L3 동적 바인딩).

| 변수 | 역할 | 현재 기본값 |
|------|------|-----------|
| `--color-primary` | 주조색 (CTA·강조) | `#6b9eff` |
| `--color-secondary` | 본문/보조 텍스트 | `#333333` |
| `--color-accent` | 포인트 | `#ff6b6b` |

### Apple 시스템 컬러 (임시 카테고리)
| 이름 | HEX |
|------|-----|
| Blue | `#007AFF` |
| Green | `#34C759` |
| Indigo | `#5856D6` |
| Orange | `#FF9500` |
| Pink | `#FF2D55` |
| Purple | `#AF52DE` |
| Red | `#FF3B30` |
| Teal | `#5AC8FA` |
| Yellow | `#FFCC00` |
| Gray | `#8E8E93` |

- 텍스트 가독성: 본문은 명도 대비 충분히 (밝은 배경엔 Gray/검정 계열).
- 한 섹션에 강조색은 1개로 절제.

## 4. 정렬

- 히어로/Hook 성격 섹션 텍스트는 **center 기본** (`build_basic_section` 기본값).
- 정보 나열·긴 본문은 `left`가 읽기 편할 수 있음 → `align` 파라미터로 override.
