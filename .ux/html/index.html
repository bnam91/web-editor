동작 방식

모드 — PROFILE(핵심작업·기준값·영역 명세 생성) / RUN(차원 병렬 감사→리포트) / MATRIX(영역×차원 격자) / GATE(머지 전 UX 사전 평가) / FIX(승인된 제안만 적용) / UPGRADE(방법론 갱신)
신뢰 장치 — ① judgment(휴리스틱 근거+제안+승인)와 conformance(기준값 판정) 트랙 분리 ② impact=high·severity=blocker/major는 독립 검증자가 적대적 반증(오적용/트레이드오프/의도된 결정 3축) ③ 의도된 UX 결정 사전 제외 + 노이즈 캡(judgment 10/conformance 15) ④ heuristic_basis 근거 추적(못 대면 no-basis) ⑤ 영역 전원 보고(침묵 금지)
3자 경계 — ux=구조적 판단("이 구조가 옳은가") / qa=실행 마찰("밟아보니 걸리나") / design=시각 이탈("기준에서 벗어났나")
