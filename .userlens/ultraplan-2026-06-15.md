# Userlens 울트라플랜 v3 — GODITOR "카테고리별 +5건" 채굴 계획 (2026-06-15)

> 목표: A 이상함 / B 기능갭 / C 버그 / D 희귀버그 **각 +5건 = 총 +20건**.
> 원칙(엄수): **억지 채움 금지.** 진짜 새 발견만. 카테고리가 5건을 정직하게 못 채우면 그 사실을 보고(가짜로 채우지 않음). 기존 트래커(A1~A34·B1~B15·C1~C20·D1~D7·E)와 2026-06-15 제외 8건은 재보고 금지.

## 0. 왜 이 계획인가 (이번 RUN이 통한 이유의 역설계)
이번에 C20(over-scroll)을 잡은 건 **"코드기능 프로브"가 아니라 "줌·스크롤·선택의 시스템 불변식을 의심"**했기 때문. 그 vein을 카테고리별로 확장하면 진짜가 더 나온다:
- **C 버그**: "보이는데 안 남는/안 나가는/안 맞는" — L2 저장재오픈·L3 export·줌좌표 불변식이 깨지는 셀. (그라데이션·C20이 다 여기서 나왔다)
- **D 희귀버그**: 정상순서엔 안 터지고 **이상한 순서**(연타·중간취소·전환중조작·autosave 경합)에서만 터지는 것.
- **A 이상함**: 처음 쓰는 사람 기준 기대-실제 갭(기본값·가시성·라벨·위계).
- **B 기능갭**: Figma/Adobe라면 당연한데 없는 것(스냅·단축키·분배·잠금 등).

## 1. 미검 매트릭스 셀 = 1순위 표적 (ultraplan v2 §2에서 ○/▢)
아래 셀은 **영역은 아는데 이 축을 한 번도 안 봤다** → 채굴 확률 최고:
- **L2 저장재오픈**: 도형·체크리스트/핀·그룹·그라데이션블록·Frame = 전부 ○ (그라데이션 클래스가 살아있을 자리)
- **L3 export**: 도형·스티커·핀·체크리스트·Frame = ○ (C18 어노테이션 누수의 형제 버그 가능)
- **이동/스냅**: 텍스트 외 거의 ○ (drag-snap 자체 미검 — B 기능갭 유력)
- **완전 사각 영역**: Templates 적용·Frame auto/free 전환·플러그인 deep

## 2. 카테고리별 채굴 표적 (각 ≥5 후보 — 통과분만 적재)

### 🔴 C 버그 (+5) — 직접 재현 필수, 4겹 오라클
| # | 표적 | 오라클 | 가설 |
|---|------|--------|------|
| C-a | 줌 변경 후 스크롤 위치/가로센터 드리프트 | 줌 40→100→Fit 반복 후 보던 위치 유지되나 | C20 형제: 줌 시 보던 콘텐츠 튐 |
| C-b | 도형/스티커/핀 **L2 저장재오픈** | 생성·편집→flushSave→reload 복원 | 그라데이션 클래스 잔존 |
| C-c | 스티커/핀/체크리스트/Frame **L3 export** | "이 섹션 내보내기" 산출 이미지 검사 | C18 형제 누수/누락 |
| C-d | 컴포넌트별 편집 렌더(Table셀·Chat·Step·Comparison·Banner·Laurel) | 각 편집 직후 + reload | 특정 컴포넌트 깨짐 |
| C-e | undo/redo 경계(다중조작 후 undo, redo 스택) | 조작→undo→redo DOM+메타 | 무손실 깨짐 |

### 🟣 D 희귀버그 (+5) — 의도적 이상순서
| # | 표적 | 시퀀스 |
|---|------|--------|
| D-a | 추가↔undo 연타 | 블록 빠르게 add×3 → 즉시 undo 연타 → 상태 정합 |
| D-b | autosave 경합 | 편집 직후 flushSave 진행 중 다시 편집/페이지전환 |
| D-c | 페이지전환↔undo↔autosave 복합 | 새 페이지 추가→블록→다른페이지→undo |
| D-d | 선택 desync | 다중선택 후 레이어에서 한 개 삭제 시 캔버스 선택 불일치 |

### 🟠 A 이상함 (+5) — 처음 쓰는 사람 기대-실제 갭
| # | 표적 |
|---|------|
| A-a | 기본 줌 40%가 가로·세로 어느 쪽도 안 맞음 — 왜 40%? 첫인상 혼란 |

| A-e | 상단 실프로젝트 탭 다수 — 테스트 중 오클릭 위험/식별 어려움(가드 안내 부재) |
| A-f | 에셋 체커보드에 "클릭해 채우라"는 명시 affordance 부재(처음 쓰는 사람 기준) |
