# 02. 브랜치 시스템

## 개요

git과 유사한 브랜치 구조로, 프로젝트의 여러 버전을 병렬로 관리할 수 있다.
각 브랜치는 독립된 캔버스 스냅샷을 가지며, 전환·병합이 가능하다.

---

## 브랜치 종류 및 색상 규칙

| 브랜치 | 색상 | 용도 |
|--------|------|------|
| `main` | 🟢 초록 | 최종 완성본, 배포 기준 |
| `dev` | 🟠 주황 | 개발·작업 중간본 |
| `feature/*` | 🔵 파랑 | 특정 섹션 실험·기능 개발 |

색상은 브랜치 패널 아이콘, 드롭다운 dot, 커밋 모달 버튼에 공통 적용된다.

---

## 저장 구조

```
projects/{id}.json
├── currentBranch: "main"
└── branches:
    ├── main:    { snapshot, updatedAt, scope }
    ├── dev:     { snapshot, updatedAt, scope }
    └── feature/헤더: { snapshot, updatedAt, scope }
```

- 브랜치 전환 시 현재 브랜치의 스냅샷을 먼저 저장한 뒤 대상 브랜치로 복원
- 자동저장(1.5초 디바운스)은 활성 브랜치 기준으로 동작

---

## 커밋 시스템

커밋은 특정 시점의 스냅샷을 히스토리로 기록하는 체크포인트다.

```
projects/{id}.json
└── commits: [
    { id, message, timestamp, branch, snapshot },
    ...
  ]
```

- 커밋은 브랜치와 무관하게 전체 히스토리에 쌓임
- 커밋 모달에서 브랜치 필터 탭으로 브랜치별 조회 가능
- 복원 시 해당 커밋의 스냅샷을 캔버스에 적용

---

## ⚠️ 알려진 충돌 시나리오 (미해결)

> 현재 충돌 감지 기능 없음. 아래 시나리오는 인지 후 추후 대응 예정.

### 1. 병합 시 덮어쓰기 (가장 위험)
```
feature에서 섹션A 수정
main에서도 섹션A 수정
→ feature → main 병합 시 main 내용이 경고 없이 사라짐
```
**현재 동작**: 소스 브랜치가 대상 브랜치를 완전히 대체
**향후 대응**: 병합 전 diff 비교 + 충돌 확인 단계 추가

---

### 2. 자동저장 타이밍 충돌
```
브랜치 전환 클릭
→ 이전 브랜치 자동저장(1.5초 딜레이) 진행 중
→ 저장 완료 전 새 브랜치 데이터가 덮어쓸 수 있음
```
**현재 동작**: 전환 시 즉시 현재 상태 저장 시도하나 비동기 타이밍 보장 안 됨
**향후 대응**: 전환 전 autoSave flush 보장

---

### 3. 커밋 복원 후 브랜치 불일치
```
커밋 복원 → 캔버스만 되돌림
→ 브랜치 store(branches 객체)는 복원 전 상태 그대로
→ 브랜치 전환 시 복원 전 내용이 나올 수 있음
```
**현재 동작**: `applyProjectData(snapshot)`만 호출, 브랜치 store 미갱신
**향후 대응**: 복원 시 브랜치 store도 함께 갱신

---

### 4. 병합 이력 미기록
```
병합 완료 → 커밋 히스토리에 기록 없음
→ 언제 어떤 브랜치를 병합했는지 추적 불가
```
**현재 동작**: 병합은 단순 스냅샷 교체, 커밋 생성 안 함
**향후 대응**: 병합 시 자동 커밋 생성 (`"Merge feature/X → main"`)

---

## main 브랜치 보호 (Protected Branch)

git의 protected branch 개념과 동일하다.
main은 항상 최종 완성본(배포 버전)을 보장하며, **병합으로만 업데이트**된다.

### 브랜치 역할 흐름

```
[dev]  → 작업
[feature/*] → 실험
         ↓ 병합
[main] ← 병합으로만 업데이트 가능 (직접 편집 불가)
         ↓ 내보내기(Export/피그마 업로드)만 가능
```

### 장점

- main = 항상 최종 완성본 보장
- 실수로 main 직접 수정하는 사고 방지
- "배포 버전" 개념이 명확해짐

### 동작 방식

- main 브랜치로 전환 시 캔버스 `pointer-events: none` 적용
- 상단에 밝은 초록 잠금 배너 표시 (focus-mode-bar 바로 아래)
- **임시 잠금 해제** 버튼으로 편집 가능 (세션 중 1회)
- 다른 브랜치로 전환 후 main으로 돌아오면 자동 재잠금

### 새 프로젝트 생성 기본값

- main + dev 브랜치 자동 생성
- 시작 브랜치는 `dev`로 설정 (main 직접 편집 방지)

### 잠금 상태 플래그

```js
let _mainUnlocked = false;
// switchBranch() 호출마다 false로 초기화됨
```
