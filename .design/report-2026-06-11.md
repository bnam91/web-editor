## A. 총평 / 위험도

**위험도: 중(MEDIUM-HIGH).** 기능 파괴는 없으나 **디자인 시스템이 둘로 갈라져 있고(토큰 이원화)**, 그로 인해 **접근성 대비 위반이 시스템 전역에 깔려 있다.** 가장 심각한 단일 사실:

> **`design-tokens.css`의 잘 설계된 2단 토큰 체계(primitive→semantic)가 사실상 사문화됐다.** 의미토큰 실사용 9~54회 vs `editor-base.css`의 `--ui-*` 877~1109회. 의미토큰 49개 중 36개(73%)가 dead — `--radius-*`, `--space-0..6`, `--color-danger/accent` 전부 포함. 같은 색이 두 이름으로 존재(`--bg-app`=`--ui-bg-app`=#1a1a1a)해 drift 위험.

이 이원화가 색·간격·radius·대비 차원의 발견 다수를 **하류에서 유발**한다. 두 번째 축은 **저대비 텍스트 토큰**(`--ui-text-dim` #555, `--text-hint` #5a5a5a)이 9~11px UI 텍스트로 ~110곳 쓰이는데 AA-large 3:1조차 미달이라는 점 — 이건 렌더로 확정했다.

## B. 확정 문제 (confirmed — 렌더 실측 또는 결정적 검증)

### 🔴 B1. 저대비 UI 텍스트 — AA 위반 [a11y / blocker]
CDP 9334 실측. 실제 렌더된 UI 크롬 요소에서 합성 배경 대비 측정:
| 토큰 | 실측 요소 | 폰트 | 배경 | 대비율 | AA 4.5 | 판정 |
|------|----------|------|------|--------|--------|------|
| `--ui-text-muted` #777 | `.section-label` "Section 01" | 11px/500 | #2a2a2a | **3.21:1** | FAIL | **확정 blocker** |
| `--ui-text-dim` #555 | `.tb-badge` | 10px/400 | #2a2a2a | **1.93:1** | FAIL | **확정 blocker** |

`--ui-text-dim`(#555)·`--text-hint`(#5a5a5a)는 모든 셸 배경(#1a~#2e) 위에서 **AA-large 3:1조차 미달**. placeholder·라벨·badge로 ~110곳 실사용. fix: 다크 UI 최소 가독 텍스트를 #888(–ui-text-sub 근처)+ 이상으로 상향, dim/hint는 비텍스트(구분선) 용도로만.

### 🔴 B2. 토큰 체계 이원화 / dead 토큰 [color + css-health / major, confirmed]
결정적 grep: 의미토큰(`--color-/--bg-/--text-/--space-/--radius-`) 정의 49개 중 **36개(73%) dead**(정의부 밖 var() 0회). primitive 17개도 연쇄 dead. 실가동은 `--ui-*`(22개 정의·1109회 참조). → 신규 UI 코드가 어느 체계를 써야 할지 모호, 같은 값 두 이름 drift. fix: 한 체계로 정본화(권장: 실사용 `--ui-*` 유지, design-tokens 의미층은 `--ui-*` alias로 재정의하거나 폐기).

### B3. focus-visible 시스템 부재 [states / major, confirmed]
CSSOM: `:focus-visible` 정의가 16개 CSS 중 `color-picker.css` **1곳뿐**. .tb-btn/.zoom-btn/#home-btn/.prop-btn/.cpm-btn/.layer-item 등 전 UI 버튼에 키보드 포커스 표시 없음 → 키보드 사용자 위치 상실(a11y). fix: color-picker:418-424 패턴을 공용 규칙 승격.

### B4. prefers-reduced-motion 전무 [motion / major, confirmed]
16개 CSS에서 `prefers-reduced-motion` 0회. UI 크롬에 무한반복 애니 6종(autosave blink, ai-spin×2, ai-fill-spin, cpmt-pulse, aig-shimmer) 상주. 전정장애 사용자 가드 없음.

### B5. --icon-* 토큰 사문화 [icon / major, confirmed]
`--icon-layer-size`/`--icon-prop-size` 등 5종 정의·**실사용 0회**. `.prop-block-icon`은 20px로 토큰값(16px)과 값 자체가 불일치. (B2와 같은 "정의는 했으나 안 씀" 패턴.)

## C. 의심 문제 (suspected — 정적 근거, 렌더 미확정)

### C1. 타이포 스케일 토큰 부재 [typo / major]
UI 크롬 font-size px 직접 405회·21개 distinct(11px 162회 최다). 색·간격·radius는 토큰화됐는데 **타이포만 누락**. 캔버스 콘텐츠는 104/72/52/36/26 6단 스케일(`--preset-*`) 보유 — UI엔 없는 비대칭. 9px 서브텍스트 36회(editor-panels 집중)는 B1 대비와 교차 → 렌더 실측 시 blocker 승격 가능.

### C2. 위험 버튼 danger 토큰 우회 [component / major]
`.prop-btn-danger`가 danger 토큰 대신 raw hex 3종(#6b2e2e/#e06060/#2a1818, editor-props:485). **v2 필수 준수 규칙(위험버튼=danger class/token) 위반.**

### C3. radius·shadow 전용 토큰 부재 [component / major]
`--radius-*`는 정의됐으나 dead(B2), `--ui-shadow-*` elevation 토큰은 아예 없음. → box-shadow 45종 난립(모달 동급 깊이만 5종+), radius off-scale 143개(3px×50/5px×43/8px×26). 같은 prop 버튼군이 border 토큰 갈림(`--ui-border-strong` vs `--ui-border-mid`).

### C4. 컨테이너/행 패딩 구조적 비일관 [spacing / major]
동일 역할 패딩이 세로 5/6/8/10·가로 8/10/12로 교차해 8종+ 변종. 4px 그리드 off-grid 총 438건(6px×146/10px×85/5px×63) — design-tokens.css:182에 **점진 이관 부채로 이미 인지됨**(per-line 버그 아님, 군집). 핫스팟 editor-panels(136)>extra(96). CDP 렌더로 행 높이 어긋남 확인 권고.

### C5. 아이콘셋 혼용 [icon / major]
UI 액션 버튼에 이모지(✨📋⬇✂️🔲)와 인라인 SVG 혼합. 같은 위계 버튼인데 ✨=OS 이모지, 형제=SVG. 닫기 글리프 ✕/× 혼용, stroke-width 15종 산재.

### C6. outline:none 미보완 2건 [states / major]
`.file-page-label`(클릭 버튼, panels:596), `.prop-slider`(range, props:61)가 border 교체 없이 focus ring만 제거. (input/select의 `outline:none + :focus{border-color}` 짝은 정당 — 제외.)

### C7~ (minor·nit, 그룹 접힘)
line-height unitless 9종+px 혼용 / font-family 스택 순서 제각각 / transition `:all` 23회 / duration ms·s 이중표기 / disabled opacity 0.3~0.7 분산 / `!important` 회피가능 ~95개(목표 114→25) / `.prop-full-btn`·`.comp-shelf-card-info` 중복 정의. **dead 셀렉터는 샘플 검증 전부 JS 사용중 → 오탐 0.**

## D. 설계/UX 관찰 (기준 부재 영역 — 취향 아님)

- **"정의했으나 안 쓰는 토큰"이 반복 패턴**: 의미토큰·radius·space·icon 토큰 모두 설계는 좋은데 코드가 안 따라옴. 토큰 추가보다 **실사용 강제(린트)**가 필요한 단계.
- **UI 크롬 vs 캔버스 콘텐츠 경계는 명확**(profile 예외 규칙대로 깔끔히 분리됨). 캔버스는 사용자값이라 불가침이 옳음 — 이 분리는 잘 지켜지고 있다.
- 프리셋 4종(default/dark/brand/minimal) 변수 키 12개 **완전 동일 — 키 누락 0건(클린).**

## 엔진 피드백 (design-squad v3.0 첫 실전 마찰)

1. **[순작동] 2층 적대적 검증이 실제로 오탐을 걸러냄** — DSN-a11y-05(#666)를 에이전트가 "캔버스일 수 있음, 표면 확인 요"로 verify-candidate로 올렸고, 중앙이 `.cvb-card-meta`=캔버스블록 확인 후 **refuted**. 무편향 게이트 설계가 의도대로 동작.
2. **[마찰] 정적 차원의 "확정" 등급 모호** — 대비는 measure로 깔끔히 confirmed 되나, "토큰 0회 사용/focus-visible 1파일" 같은 **결정적 grep 사실**이 measure도 render도 아닌 pattern인데 confidence는 confirmed여야 맞음. 엔진에 `evidence_grade: deterministic`(또는 pattern이되 confirmed 허용 명문화) 추가 제안. 현재는 pattern+confirmed로 표기.
3. **[마찰] 렌더 미히트 처리 규약 부재** — 화면에 현재 안 뜬 요소(다른 패널)는 confirmed 승격 불가. "어느 화면을 띄워 무엇을 측정하는지" 렌더 시나리오를 프로파일에 명시하는 칸이 있으면 좋음(qa-squad의 flows/ 처럼). 지금은 커버리지 구멍으로 처리.
4. **[마찰] 토큰 이원화가 5개 차원에 중복 출현** — color/css-health/a11y/component/icon이 같은 뿌리(정의했으나 안 씀)를 각자 보고. 차원 교차 발견을 중앙이 1개 root-cause로 머지하는 단계가 있으면 리포트 중복이 줄겠음. 이번엔 수동 머지(B2).
5. **[순작동] 의심 노이즈 캡/그룹핑이 spacing(438건)·color(352 hex)에서 제대로 작동** — per-line 폭주 없이 군집 2건으로 접힘. 부채 인지 예외 주입(px 3078)이 효과적.
6. **[제안] 프로파일 active_dimensions 9개는 첫 실전엔 적정**했으나, 정적 전용 환경이면 a11y/states/motion은 렌더 필수라 "CDP 없으면 전량 suspected" 경고를 PROFILE 단계에서 더 강하게 띄울 것.
